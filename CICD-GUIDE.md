# CI/CD Pipeline: Complete Guide for Live Demo

## Architecture (Single EC2 Instance)

```
┌─────────────────────────────────────────────────────┐
│                    EC2 Instance                       │
│                                                      │
│  ┌──────────┐         ┌──────────────────────┐      │
│  │  Nginx   │───/───▶ │  React App (dist/)   │      │
│  │ (Port 80)│         │  /var/www/html/       │      │
│  │          │         └──────────────────────┘      │
│  │          │                                        │
│  │          │──/api/─▶ ┌──────────────────────┐     │
│  │          │          │  Express Server       │     │
│  └──────────┘          │  (Port 3001, PM2)    │     │
│                        │  + SQLite Database    │     │
│                        └──────────────────────┘     │
└─────────────────────────────────────────────────────┘
            ▲
            │ HTTP (port 80)
            │
      Users access via EC2 Public IP
```

## CI/CD Flow

```
Developer pushes code to GitHub (main branch)
        │
        ▼
GitHub Actions triggers automatically
        │
        ▼
Runner SSHs into EC2 instance
        │
        ▼
Pulls latest code → Builds frontend → Restarts backend
        │
        ▼
App is live with new changes (< 2 minutes)
```

## PART 1: Manual Deployment on EC2 (First Time)

### Step 1: Launch EC2 Instance

1. Go to **AWS Console → EC2 → Launch Instance**
2. Settings:
   - **Name**: hackathon-app
   - **AMI**: Ubuntu 22.04 LTS (Free tier eligible)
   - **Instance type**: t2.micro (Free tier)
   - **Key pair**: Create new → name it `hackathon-key` → Download `.pem` file
   - **Security Group**: Create new with these rules:

| Type | Port | Source | Purpose |
|------|------|--------|---------|
| SSH | 22 | My IP | For you to connect |
| HTTP | 80 | 0.0.0.0/0 | For users to access app |
| Custom TCP | 3001 | 0.0.0.0/0 | Backend API (optional, Nginx proxies it) |

3. Click **Launch Instance**

### Step 2: Connect to EC2

```bash
# Make key file secure (required by SSH)
chmod 400 hackathon-key.pem

# Connect to your EC2 instance
ssh -i hackathon-key.pem ubuntu@YOUR_EC2_PUBLIC_IP
```

### Step 3: Run Initial Setup Script

Once connected to EC2:
```bash
# Download and run the setup script (or copy-paste it)
# First, clone your repo manually:
cd ~
git clone https://github.com/YOUR_USERNAME/hackathon.git
cd hackathon

# Run the setup script
bash scripts/ec2-initial-setup.sh
```

### Step 4: Verify It Works

Open browser → Go to `http://YOUR_EC2_PUBLIC_IP`

You should see your React app! Try:
- `http://YOUR_EC2_PUBLIC_IP` → React frontend
- `http://YOUR_EC2_PUBLIC_IP/api/holidays` → API response (JSON)

## PART 2: Set Up CI/CD (Auto Deploy on Push)

### Step 1: Push Code to GitHub

```bash
# On your local machine
cd /home/user/Documents/Pi_Erp_project/hackathon
git init
git add .
git commit -m "Initial commit with CI/CD pipeline"
git remote add origin https://github.com/YOUR_USERNAME/hackathon.git
git push -u origin main
```

### Step 2: Add GitHub Secrets

Go to: **GitHub Repo → Settings → Secrets and variables → Actions → New repository secret**

Add these 2 secrets:

| Secret Name | Value | How to get it |
|-------------|-------|---------------|
| `EC2_HOST` | `13.235.x.x` | EC2 Dashboard → your instance → Public IPv4 address |
| `EC2_SSH_KEY` | Contents of `.pem` file | Open `hackathon-key.pem` in text editor, copy ALL content |

To copy the .pem content:
```bash
cat hackathon-key.pem
# Copy everything from -----BEGIN RSA PRIVATE KEY----- to -----END RSA PRIVATE KEY-----
```

### Step 3: Test the Pipeline

```bash
# Make a small change (e.g., edit a component)
git add .
git commit -m "Test CI/CD pipeline"
git push origin main
```

Then go to **GitHub → Actions tab** → Watch the pipeline run!

## Files Created & Their Purpose

```
hackathon/
├── .github/
│   └── workflows/
│       └── deploy.yml          ← CI/CD PIPELINE (GitHub Actions)
├── scripts/
│   ├── ec2-initial-setup.sh    ← ONE-TIME EC2 SETUP (run once)
│   └── nginx.conf              ← NGINX CONFIG (serves React + proxies API)
├── server/
│   └── index.js                ← EXPRESS BACKEND (runs on port 3001)
├── src/                        ← REACT FRONTEND CODE
├── dist/                       ← BUILT FRONTEND (generated by npm run build)
└── CICD-GUIDE.md               ← THIS FILE (documentation)
```

### File Explanations for Demo:

| File | What to say in demo |
|------|-------------------|
| `deploy.yml` | "This is our pipeline definition. GitHub reads this file and knows what to do when we push code." |
| `ec2-initial-setup.sh` | "This is our infrastructure setup script. It installs Node.js, Nginx, PM2, and configures everything on a fresh EC2 instance." |
| `nginx.conf` | "Nginx acts as a reverse proxy. It serves our React app on port 80 and forwards any /api/ requests to our Express backend on port 3001." |

## How to Explain in Live Demo (Script)

### Opening (30 seconds)
> "I'll show you how code goes from my laptop to a live server automatically. This is CI/CD — Continuous Integration and Continuous Deployment."

### Show the Architecture (1 minute)
> "We have one EC2 instance running everything:
> - **Nginx** on port 80 serves our React frontend
> - **Express** on port 3001 handles our API
> - **PM2** keeps the backend alive 24/7
> - **Nginx reverse proxy** routes /api/ calls to Express
>
> Users hit the public IP → Nginx decides: is it a page? Serve React. Is it /api/? Forward to Express."

### Show the Pipeline File (1 minute)
> "Here's our `deploy.yml`. When I push to main:
> 1. GitHub spins up a clean Ubuntu machine
> 2. It SSHs into our EC2 using a private key stored in GitHub Secrets
> 3. Pulls the latest code
> 4. Rebuilds the frontend
> 5. Copies built files to Nginx's directory
> 6. Restarts the backend with PM2
>
> Total time: about 90 seconds. Zero manual work."

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Can't SSH into EC2 | Check security group allows port 22 from your IP |
| Website not loading | Check security group allows port 80 from 0.0.0.0/0 |
| API returning errors | SSH in, run `pm2 logs hackathon-backend` |
| GitHub Action fails at SSH | Verify EC2_HOST is the public IP, EC2_SSH_KEY is complete .pem content |
| Old version showing | Hard refresh browser (Ctrl+Shift+R) or wait 30 seconds |
| PM2 not starting | Run `pm2 delete all` then `pm2 start index.js --name hackathon-backend` |

## Useful Commands on EC2

```bash
# Check if everything is running
pm2 status                          # Backend status
sudo systemctl status nginx         # Nginx status

# View logs
pm2 logs hackathon-backend          # Backend logs
sudo tail -f /var/log/nginx/error.log  # Nginx error logs

# Restart services
pm2 restart hackathon-backend       # Restart backend
sudo systemctl restart nginx        # Restart nginx

# Manual deploy (if CI/CD isn't set up yet)
cd ~/hackathon
git pull origin main
npm ci && npm run build
sudo cp -r dist/* /var/www/html/
cd server && npm ci --production
pm2 restart hackathon-backend
```
